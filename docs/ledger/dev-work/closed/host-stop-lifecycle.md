# Host-Initiated Run Stop

## Scope

Implement graceful host-initiated stop for live and fixture runs in the
production layer.

This work adds:

- a host-owned stop path for fixture and process ingress
- a synchronous SDK stop surface
- profile-level `max_duration` / `max_events` bounds
- scaffolded Ctrl-C wiring that requests graceful stop

This branch does not change kernel semantics, replay format, or the
driver protocol version.

## Current State

Today:

- process drivers end only when the driver emits `{"type":"end"}` or
  the process dies
- fixture runs end only when the fixture exhausts
- Ctrl-C kills the process before host finalization writes a capture
- there is no canonical bounded-run surface (`run for 15m`, `run for
  100 events`)
- the governing decision is recorded in
  `docs/ledger/decisions/host-stop-lifecycle.md`

## Design Constraints

- This branch stays in host + loader + SDK. No kernel changes.
- `max_events` counts committed events, not raw driver frames.
- Same-ingestion replay semantics remain unchanged.
- Driver protocol remains unidirectional in v0; host stop may kill the
  process driver rather than negotiate shutdown.

## What This Branch Enables

After this branch lands:

1. Long-running profiles can be stopped cleanly without losing capture
   artifacts.
2. Projects can run for bounded time or bounded event counts through
   canonical product surfaces.
3. Real live ingress/egress smoke runs can finish with replayable
   evidence instead of ad hoc channel-owned timers.
